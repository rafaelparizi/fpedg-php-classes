Introdução ao Desenvolvimento Backend

Turma 30: O Ambiente de Desenvolvimento

Para programar em PHP, precisamos de um "servidor" em nossa máquina.

  • Usaremos o XAMPP (ou similar: WAMP, MAMP).
  • Apache: O programa que atua como Servidor Web.
  • MySQL: O programa do Banco de Dados.
  • PHP: A linguagem que processa a lógica.

Ferramenta: Painel de Controle do XAMPP

É aqui que iniciamos e paramos os serviços Apache e MySQL.

Turma 30: Sintaxe Básica do PHP

O código PHP é inserido em blocos dentro do HTML.


<?php
    // Comentário de uma linha
    /* Comentário de múltiplas linhas */

    // Comandos terminam com ponto e vírgula
    echo "Olá do PHP!";
?>

Turma 30: Variáveis e Tipos de Dados

Variáveis guardam informações. Em PHP, começam com `$`. 


<?php
$texto = "Sou uma string";      // String (texto)
$ano = 2025;                    // Integer (número inteiro)
$preco = 49.99;                 // Float (número com decimal)
$disponivel = true;             // Boolean (verdadeiro/falso)

echo "O preço do produto é R$ " . $preco; // O "." concatena (junta) textos
?>

Turma 31: O Modelo Cliente-Servidor

A web funciona com base em requisições e respostas entre o seu navegador (cliente) e um computador remoto (servidor).

Turma 31: Verbos HTTP - A Intenção do Pedido

Cada requisição HTTP tem um "verbo" que indica sua finalidade.

  • GET: Usado para solicitar/buscar dados (ex: acessar um site).
  • POST: Usado para enviar/criar dados (ex: enviar um formulário de login).

Criação de Banco de Dados com MySQL

Linguagem SQL: DDL vs. DML

O SQL tem "sub-linguagens" para diferentes tarefas:

  • DDL (Data Definition Language): Para definir a estrutura (`CREATE`, `ALTER`, `DROP`).
  • DML (Data Manipulation Language): Para manipular os dados (`INSERT`, `UPDATE`, `DELETE`, `SELECT`).

Ferramenta: phpMyAdmin

É uma interface web que vem com o XAMPP para gerenciar o MySQL. Aqui executamos nossos comandos SQL.

Exemplo DDL: Criando a Estrutura

Comando `CREATE` para definir nossa tabela de usuários.


CREATE TABLE usuarios (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nome VARCHAR(100) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    senha VARCHAR(255) NOT NULL,
    data_cadastro TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Exemplo DML: Inserindo e Selecionando Dados

Comandos `INSERT INTO` e `SELECT` para adicionar e ver os dados.


-- Inserindo um novo usuário
INSERT INTO usuarios (nome, email, senha) 
VALUES ('Ana Silva', 'ana@email.com', 'senha123');

-- Selecionando todos os usuários para visualização
SELECT * FROM usuarios;

Conexão do PHP com o Banco de Dados

MySQLi: A Ponte entre PHP e MySQL

Vamos usar a abordagem Orientada a Objetos do MySQLi, que é mais moderna e organizada.

`$conexao = new mysqli();`

O Script de Conexão (`conexao.php`)

Um arquivo central para guardar as credenciais e criar a conexão.


<?php
$servidor = "localhost";
$usuario_db = "root";
$senha_db = "";
$banco = "minha_loja";

$conexao = new mysqli($servidor, $usuario_db, $senha_db, $banco);

if ($conexao->connect_error) {
    die("Falha na conexão: " . $conexao->connect_error);
}
?>

Executando uma Consulta e Lendo os Resultados

Após conectar, podemos enviar comandos SQL e ler as respostas.


<?php
require 'conexao.php';

$sql = "SELECT id, nome, email FROM usuarios";
$resultado = $conexao->query($sql);

if ($resultado->num_rows > 0) {
    while($linha = $resultado->fetch_assoc()) {
        echo "ID: " . $linha["id"]. " - Nome: " . $linha["nome"]. "<br>";
    }
} else {
    echo "Nenhum usuário encontrado.";
}
$conexao->close();
?>

Formulário de Login e Autenticação

O que são Sessões HTTP?

A solução para que o servidor "lembre" de um usuário entre diferentes páginas.

  • `session_start()`: Inicia ou resume uma sessão.
  • `$_SESSION`: Um array global para guardar informações do usuário logado.

Segurança: Nunca Armazene Senhas em Texto Puro!

A solução é usar Hashing, um processo que torna a senha indecifrável.

  • `password_hash()`: Para criar o hash ao cadastrar um usuário.
  • `password_verify()`: Para comparar a senha do login com o hash salvo no banco.

Segurança: Prevenindo SQL Injection

Um ataque onde um invasor insere comandos SQL maliciosos em um formulário. A solução é usar Prepared Statements.

  1. `prepare()`: Prepara a query com placeholders (`?`).
  2. `bind_param()`: Vincula as variáveis do usuário de forma segura.
  3. `execute()`: Executa a query.

Script de Autenticação SEGURO

Juntando Sessões, `password_verify()` e Prepared Statements.


<?php
session_start();
require 'conexao.php';

$email = $_POST['email'];
$senha_form = $_POST['senha'];

$stmt = $conexao->prepare("SELECT id, nome, senha FROM usuarios WHERE email = ?");
$stmt->bind_param("s", $email);
$stmt->execute();
$resultado = $stmt->get_result();

if ($resultado->num_rows === 1) {
    $usuario = $resultado->fetch_assoc();
    
    if (password_verify($senha_form, $usuario['senha'])) {
        $_SESSION['usuario_id'] = $usuario['id'];
        header("Location: painel.php");
        exit();
    }
}
echo "Email ou senha inválidos.";
?>

Avaliação Prática e Revisão

O Fluxo Completo da Autenticação

Login -> Autenticação -> Criação da Sessão -> Acesso a Páginas Restritas -> Logout -> Destruição da Sessão.

Protegendo uma Página (Análise do Código)

O `painel.php` verifica se a sessão existe antes de mostrar o conteúdo.


<?php
session_start();

// Se a variável 'usuario_id' NÃO FOI DEFINIDA na sessão...
if (!isset($_SESSION['usuario_id'])) {
    // ...redirecionamos para o login e paramos o script.
    header("Location: login.html");
    exit();
}

// Se o script continuar, o usuário está logado!
echo "
Bem-vindo ao seu painel!
";
?>

Saindo do Sistema (Análise do Logout)

O `logout.php` faz a limpeza e destrói a sessão.


<?php
session_start();
$_SESSION = array(); // Limpa as variáveis
session_destroy(); // Destrói a sessão
header("location: login.html");
exit;
?>
